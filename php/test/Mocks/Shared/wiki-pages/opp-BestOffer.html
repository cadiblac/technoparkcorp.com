Vision

Statement: Nowadays the amount of different devices installed and used in a typical living house is growing. At least every modern house has a TV, a few computers, a few laptops, sound system with home cinema, lights control system, and security system. Sooner or later every component will provide a digital interface to its functionality, through Wi-Fi, Bluetooth or some other interface. There is a market opportunity to give end user the ability to control all devices through one simple and intuitive interface, preferably through TV panel. The customer is trying to pursue this opportunity and develop such an integrating system. Moreover, with the fast growing of Internet and its presence in almost every house there is an opportunity to interconnect such an integrating system with a central server and make it networking instead of standalone.

Product: This SUD is a software component installed on one of home computers, and connected to TV. The SUD acts as a mediator between end user and all other components. The user communicates with them through one simple and unified user interface. Every component is represented in the SUD by its "Gadget" or "Application" (Gadget's viewpoint is limited and fixed, while Application has bigger space and more flexibility in interaction with end user). The SUD when working becomes a network element of a web system, with a center at ruzoid.com.

Actors:

  • Householder: a human being managing the system in his own (or rented) house, also known as "administrator". Householder is a User as well and all functionality available for a User is available for a Householder.
    • F1: "Manage Gadgets" Householder can install, configure and remove gadgets and applications; can grant/revoke permissions to other users for particular gadgets and applications; can get access to "security zone" of every gadget and application.
    • F2: "Manage Users" Householder can create, delete, change/configure other users; can monitor protocols of users activity; can block certain types of access for particular users.
  • User: a regular user of the system, probably one of Householder's relatives or guests
    • F3: "Play with Gadgets" User can start and stop any Application; can add and remove certain Gadgets to his "panel"; can play with gadgets and applications according to the rules/terms of their usage.
  • Gadget: a software component provided by a third-party or by the customer, that comply to some requirements for its application programming interface (API). Gadget and Application are almost the same type of an actor, but they differ in their user interface capabilities.
    • F4: "Perform" Gadget can interact with a User through the SUD, sending to us its output and receiving back users' input.
  • Server: a web service provider hosted at ruzoid.com
    • F5: "Web Service" Server can receive logs from SUD, keep them in secure place, and return on demand; can authenticate users and authorize their access requests; can "rent" Applications and Gadgets to the SUD; can send emails to users through built-in SMTP server.

Quality:

  • Attractiveness: The user interface of the SUD has to be comfortable for a modern user, used to work with iPhone, iPad and similar devices. It is desired to have an ability to use the majority of functions without keyboard or mouse. Alternatives are touch-screen, a simple few-buttons TV remote control, or something like Apple remote control.
  • Interoperability: The SUD has to work both on Windows and Mac OS platforms. The Customer proposed to use Silverlight and Managed Extensibility Framework (MEF) as a software platform. The SUD has to have a properly documented interface available to third-party developers, since they might be interested to develop their own Gadgets and Applications (most possible they will be manufacturers of house equipment). If Silverlight is selected as a target software platform, all Gadgets and Applications will have to be developed in Silverlight, which imply certain limitation on marketing of the entire product.

SAO

Components:

  • Gadgets: collection of Silverlight/MEF gadgets stored locally, but originally retrieved from the Server.
  • Front: set of UI components interacting with the end-user
  • Dispatcher: controller, interacting with Front, Gadgets and Server
    • Gadgets: installing, configuring, rendering
    • Front: collecting inputs, producing outputs

Concepts:

  • MVC: There is a good situation to apply a classic Model-View-Controller design pattern, where Model is a collection of Gadgets and Applications, View is Front, and Controller is a Dispatcher. Thus we have a strict separation of Front and Gadgets.
